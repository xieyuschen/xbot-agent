# 高级技巧

xbot 高级使用模式和高级用户技巧。

## 对话

### 频繁使用 `/clear`

长对话会填满上下文窗口。切换到新任务时，运行 `/clear` 从头开始。Agent 的记忆系统会跨会话保留重要上下文。

### 压缩而非清除

如果你想保留对话但减少 token 用量，使用 `/compress`。它会总结较早的消息并释放上下文空间，不丢失线索。

### 用 context_edit 精确清理

`context_edit` 工具可以删除对话中特定的轮次或消息——适合移除不再相关的大型工具输出。

## 模型管理

### 按会话切换模型

使用 `Ctrl+N` 仅切换当前会话的模型——不影响其他会话。简单任务用便宜模型，复杂任务用强力模型。

### 多订阅

为不同提供商创建订阅，用 `Ctrl+N` 打开 LLM 面板切换：
- 一个用于日常工作（均衡成本/质量）
- 一个用于复杂推理（高端模型）
- 一个用于快速查找（快速/便宜模型）

### 子 Agent 模型层

在 `/settings` 中配置 Vanguard / Balance / Swift 层，让子 Agent 自动为不同复杂度的任务选择合适的模型。

## 子 Agent

### 委派给专门化 Agent

不要什么都自己做。委派专注任务：
- `explore` — 代码库探索和分析
- `~/.xbot/agents/` 中的自定义角色 — 你自己的专门化角色

### 用群聊做架构决策

使用会议模式获取多个专家视角：
```
"评审这个 API 设计。分别获取安全、性能和 UX 专家的意见，然后综合。"
```

## 自动化

### 用 Cron 定时任务

Agent 可以为自己调度任务：
```
"每天早上 9 点检查 CI 状态，有失败就通知我。"
```

### 长时间运行的后台任务

为构建过程、开发服务器和测试使用后台模式：
```
"在后台运行测试套件并告诉我结果。"
```

Agent 给你一个任务 ID——用 `task_status` 查看进度，不会阻塞对话。

## 飞书高级功能

### 交互式消息卡片

Agent 可以创建丰富的飞书卡片，包含按钮、图片和交互元素。只需描述你想要的：
```
"创建一个飞书卡片，展示项目状态，每个部分有可点击的按钮。"
```

### 多维表格自动化

读取、写入和管理飞书多维表格：
```
"读取项目追踪表，创建一个逾期任务摘要。"
```

### 飞书文档

以编程方式创建和编辑飞书文档：
```
"创建一个包含今天行动项的会议纪要文档。"
```

## 配置

### AI 自助配置

Agent 可以调整自己的设置——无需手动编辑配置：
- "切换深色主题"
- "把最大并发设为 5"
- "把字体大小设为大"
- "把侧边栏宽度设为 40 字符"

### 自定义主题

在 `~/.xbot/themes/` 中创建自定义主题 JSON 文件。让 Agent：
"创建一个暖色调的自定义主题。"

### 常用工作流的技能

为你重复的工作流创建技能（`~/.xbot/skills/` 中的 Markdown 文件）：
- GitHub PR 审查技能
- 部署清单技能
- 代码风格指南技能

{{< hint type=tip >}}
**最好的技巧：** 直接让 Agent 做事。xbot 设计为对话式——如果你能描述你想要的，Agent 通常能想出怎么做。
{{< /hint >}}
